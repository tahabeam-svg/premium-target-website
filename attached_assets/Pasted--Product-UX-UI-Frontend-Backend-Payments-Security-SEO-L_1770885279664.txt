10) الدفع Paymob + Webhooks + أمان + Sandbox/Prod
=====================================================
- PAYMOB_ENV=sandbox|production
- Webhook HMAC verification + idempotency
- لا تعتبر الطلب “Paid” إلا بعد webhook success

حماية إضافية للكوبونات (Anti-bruteforce):
- rate limiting أقوى على محاولات إدخال الكوبون
- captcha تدريجي عند كثرة المحاولات

Prisma Models (مطلوبة):
- Order: status(draft/initiated/paid/failed/canceled/fulfilled/refunded), orderNumber, lang, currency(SAR),
  subtotalSnapshot, vatSnapshot, discountSnapshot, totalSnapshot, customerName, customerEmail, customerPhone, brief, couponCode?
- OrderItem: refId, type(package/addon/custom), titleSnapshot, unitPriceSnapshot, qty, lineTotalSnapshot
- Payment: provider, paymobReference, transactionId UNIQUE, status, rawPayload
- CouponRedemption: couponCode, customerKey(emailOrPhone), orderId, status(reserved/confirmed/expired), timestamps
- Receipt: orderId UNIQUE, receiptNumber, issuedAt, totals snapshots, itemsSnapshot(JSON)
- WebhookLog: signatureOk, payload, createdAt
- ErrorLog: scope + orderNumber? + payload?

Endpoints:
1) POST /api/checkout/create
- يجلب الأسعار من Sanity
- يحسب subtotal/VAT/discount/total
- يخزن snapshots في Order/OrderItem
- يطبق coupon داخل DB transaction (atomic) ويمنع السباقات
- ينشئ Paymob checkout_url ويرجعه

2) POST /api/webhooks/paymob
- تحقق HMAC + سجل WebhookLog
- Idempotent (لا تكرر paid/receipt/emails عند تكرار transactionId)
- success:
  - Order=paid, Payment=success
  - CouponRedemption: reserved → confirmed
  - Receipt من snapshots
  - Emails: للعميل + إشعار داخلي
- failed/canceled:
  - تحديث الحالة
  - CouponRedemption: reserved → expired

3) GET /api/orders/[orderNumber]
- يعيد حالة الطلب للواجهة (thank-you/pending)

19) Coupon Lifecycle + TTL (إلزامي)
=====================================================
- reserved → confirmed فقط بعد webhook success
- reserved → expired عند failed/canceled
- إذا بقي reserved أكثر من 30 دقيقة دون نجاح دفع: تتحول تلقائيًا expired (cleanup job/cron أو عند أي محاولة جديدة)
